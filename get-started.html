Get started

How to use

Everything is based on wrappers (it will generate iframe for itself) and you can configure this as your page requires. id attribute and ps-wrapper class is always required for working. When your page is loaded the anchors what are targeted for current wrapper are getting a ps-active class.

<div id="sample" data-ps-autoload="true" data-ps-src="default-page.html" class="ps-wrapper"></div>

Short API

id (required)

Create an instance with that id and generate an iframe for it. You can get access to the content with the url.

E.g.

Url: #/wrapper-id » Load content with default src

Url: #/wrapper-id/source » Load content with source's page body.

data-ps-src ('')

Default source for this wrapper if the source is not specified from url.

data-ps-autoload (false)

Load the default src if dom is copied from the iframe (domcontentloaded).

data-ps-fixed (true)

If the url changes the content can be removed or not.

data-ps-animation (slide-down)

The animation name which is declared in css (e.g. Demo page)

data-ps-duration (1)

Animation duration

data-ps-inactivate (true)

The wrapper get an inactivate class while it is under animation. (For css3 fany loading)

Custom events on ps-wrapper

ps:load

Triggered when iframe is loaded and animation is executed.

ps:render

Triggered when content is rendered to the main window.

Recommended default styles

.ps-wrapper {
